Error State

This zero state can be used to indicate a load failure or other general error condition which prevents view readiness. It can be used to expose specific error details if a well-formed Error object or an AxiosResponse object is provided to <al-view-helper>'s [error] input (rather than merely a string, although either will work). In the case where a detailed error object is provided, those details will be exposed via an expandable/collapsible "Show Details" div.

The error state also supports retry functionality -- provide a callable to the component's [retryHandler] input, and the error screen will display a "Retry" button that will let you fire up the view initialization code again.
