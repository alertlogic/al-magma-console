Deployments are a list of assets from your appliances, agents, hosts, and collectors from your environments. You can deploy from Amazon Web Services and Microsoft Azure environments, and configure your data center deployments. When you create or edit a deployment, you must choose a level of protection.

Configuration Workflow

The most common workflow to configure a deployments is:
language
Add a Deployment
When you add a deployment, you must specify the deployment type. Some deployment types require third-party access and additional steps. Follow the tutorial in that deployment page to configure your deployments.
Discovery Assets
If you are configuring from a cloud environment, Alert Logic automatically discovers your assets when you grant third-party access. Alert Logic displays the assets discovered in your account in topology diagrams.
open_in_browser
External Assets
If all your assets were not discovered, you can add external assets manually by domain name or IP address.
Scope of Protection
You can define the scope of your protection per network basis. Each network appears within its protected region. Click a region or individual network to set the scan level or leave it unprotected.
remove_from_queue
Exclusions
You have the option to exclude assets or tags from external scanning, internal scanning, and Network IDS. You can remove an asset from exclusion at any time.
today
Scheduling
Alert Logic automatically performs certain scans. You can schedule when you want Alert Logic to scan for new assets and perform vulnerability scans. You can opt to scan on specific hours or days.
